# HTML Training

The next HTML snippets are examples of an accessible and SEO friendly semantic code, there are other ways to achieve the same results. These are based on the of the [WHATWG Living Standard](https://html.spec.whatwg.org/), best practices conventions and our dev experience dealing with UI hacks.

This HTML guide is intended to be a quick reference on your development journey, for a deeper understanding we highly recommend [MDN Web Docs](https://developer.mozilla.org/).

So you know at Z1 we encourage designers and developers to use native elements as they're meant to be. The Web is accessible by default, let's not break it!

## Links

**Internal link**

```html
<a href="/about/">About us</a>
```

**Anchor links**

```html
<a href="#section-name">Section name</a>
```

```html
<a href="#main-content" class="visually-hidden">Skip navigation</a>
```

**External links**

```html
<a href="https://blog.my-domain.com" rel="noopener noreferrer" target="_blank">
  Our blog
</a>
```

```html
<a
  href="https://marketing-partner.com"
  rel="noopener noreferrer"
  target="_blank"
>
  Marketing partner
</a>
```

```html
<a
  href="https://github.com/"
  rel="noopener noreferrer nofollow"
  target="_blank"
>
  GitHub
</a>
```

**Download link**

```html
<a href="./doc.pdf" download="file-name.pdf">Download doc</a>
```

**Image links**. If we wrap an image with a link this should have a `title` with information about the link not the image.

```html
<a href="/" title="Home">
  <img src="./header-logo.png" alt="" />
</a>
```

**Link with icon**. Any `svg` icon that is decorative or his meaning is redundant should have `aria-hidden` to no disturb the user experience and `focusable="none"` to prevent interact with this element.

```html
<a href="/post">
  Next post
  <svg aria-hidden="true" focusable="none"></svg>
</a>
```

**Icon links**. If the link doesn't have a text we need to include it as `title`.

```html
<a href="#main-content" title="Back to top">
  <svg aria-hidden="true" focusable="none" width="100" height="100">
</svg>
</a>
```

**Descriptive links**. The text should provide information about the link, if not we can add a `title` as description.

```html
<a href="#main-content" title="New features available">
  Learn more
</a>
```

## Landmarks

Use the appropriate landmarks to identify the different regions of content on a web page. The most important landmark roles are `main` and `navigation`, as nearly every page will include at least those regions.

Use HTML5 sectioning elements that have a default ARIA landmark role: `main (main)`, `nav (navigation)`, `aside (complementary)` and in some situations `header (banner)` and `footer (contentinfo)`. Do not nest landmarks with the same role.


### Banner

```html
<header>Header</header>
```

### Navigation

```html
<nav>Navigation</nav>
```

### Main

```html
<main>Main</main>
```

### Complementary

```html
<aside>Aside</aside>
```

### Contentinfo

```html
<footer>Footer</footer>
```

### Region

An alternative landmark can be created using the `section` element, which has a default landmark role of region, with an accessible name using `aria-labelledby` to reference a heading element

```html
<section aria-labelledby="this-section">
  <h2 id="this-section">Section as landmark</h2>
</section>
```

Too many landmark roles create "noise" in screen readers, making it difficult to understand the overall layout of the page.

So if the section it's not going to be a landmark, what if we use `div` instead, is that wrong? not at all, it would be the same, just that with `section` the code look nicer :)

```html
<div>
  <h2>Section title</h2>
</div>
```

```html
<section>
  <h2>Section title</h2>
</section>
```

```html

    <nav>
      <a href="/" title="Home">

      <a href="#main-content" class="visually-hidden">Skip navigation</a>
      <ul>
        <li>
          <p><a href="/page/">Page</a></p>
          <ul>
            <li><a href="/page/one/">One</a></li>
            <li><a href="/page/two/">Two</a></li>
            <li><a href="/page/three/">Three</a></li>
            <li>
              <a href="/page/four/">Second level page</a>
              <ul>
                <li><a href="/page/four/a">A</a></li>
                <li><a href="/page/four/b">B</a></li>
                <li><a href="/page/four/c">C</a></li>
              </ul>
            </li>
          </ul>
        </li>
        <li><a href="/about/">About us</a></li>
        <li><a href="/blog/">Blog</a></li>
      </ul>
      <button type="button" onclick="openModal()">Log in</button>
    </nav>
  </div>
</header>
```

### Breadcrumb

```html
<nav aria-label="Breadcrumb">
  <p>
    <a href="/blog/">Blog</a>
    >
    <a href="/blog/post-title/" aria-current="page">Post title</a>
  </p>
</nav>
```
